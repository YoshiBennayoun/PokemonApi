Pokédex API

Documentation complète de l'API RESTful Pokédex

📍 Base URL

http://localhost:3000/api

🔐 Authentification

JWT Bearer Token (24h)

📝 Format

JSON

🔐 Authentification

Important : Tous les endpoints (sauf login) nécessitent un JWT token valide dans l'en-tête Authorization.

En-tête Authorization 

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Le token est obtenu via l'endpoint de login et est valide pendant 24 heures.

📚 Endpoints

POST /api/login

Authentifie un utilisateur et retourne un JWT token.

Paramètres

Nom Type Description
usernameRequis string Nom d'utilisateur
passwordRequis string Mot de passe

Requête

{
  "username": "pikachu",
  "password": "123456"
}

Réponse (200)

{
  "message": "Connexion réussie",
  "user": {
    "id": 1,
    "username": "pikachu"
  },
  "token": "eyJhbGciOiJIUzI1NiIs..."
}

Codes d'erreur

404 - Utilisateur non trouvé

401 - Mot de passe incorrect

500 - Erreur serveur

GET /api/pokemons

Récupère la liste des pokémons avec filtrage optionnel.

Paramètres de requête

Nom Type Description
nameOptionnel string Filtre par nom (minimum 2 caractères)
limitOptionnel number Limite les résultats (défaut: 5)

Requête

GET /api/pokemons?name=pikachu&limit=10
Authorization: Bearer <token>

Réponse (200)

{
  "message": "La liste des pokémons...",
  "data": [
    {
      "id": 25,
      "name": "Pikachu",
      "hp": 35,
      "cp": 55,
      "picture": "url...",
      "types": ["electric"],
      "created": "2024-01-15"
    }
  ]
}

Codes d'erreur

400 - Terme de recherche trop court (< 2 caractères)

401 - Token invalide ou expiré

POST /api/pokemons

Crée un nouveau pokémon.

Paramètres

Nom Type Description
nameRequis string Nom du pokémon
hpRequis number Points de vie
cpRequis number Combat Power
pictureOptionnel string URL de l'image
typesOptionnel array Types (electric, water, etc)

Requête

{

  "cp": 55,
  "picture": "url...",

}

Réponse (201)

    "id": 25,
    "name": "Pikachu",
    "hp": 35,
    "cp": 55,
    "picture": "url...",
    "types": ["electric"]
  }
}

Codes d'erreur

400 - Données manquantes ou invalides

401 - Non autorisé

💡 Exemples Pratiques

1. Authentification complet 

// 1. Se connecter
const loginResponse = await fetch('http://localhost:3000/api/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'pikachu',
    password: '123456'
  })
});

const { token } = await loginResponse.json();

// 2. Utiliser le token
const response = await fetch('http://localhost:3000/api/pokemons', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const data = await response.json();
console.log(data);

2. Rechercher un pokémon 

curl -X GET 'http://localhost:3000/api/pokemons?name=pika&limit=10' \
  -H 'Authorization: Bearer YOUR_TOKEN'

3. Créer un pokémon

curl -X POST 'http://localhost:3000/api/pokemons' \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "Pikachu",
    "hp": 35,
    "cp": 55,
    "types": ["electric"]
  }'
